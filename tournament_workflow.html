BYE — Odd Players Handle karna

Agar players count 2 ki power nahi hai (e.g. 6 players), to bracket size 8 hogi. 2 "BYE" slots add hote hain — jin players ko BYE milta hai wo automatically next round mein chale jaate hain bina khele.

Bracket Advancement Formula

nextMatchNumber = ceil(currentMatchNumber / players_per_match)
nextRound       = currentRound + 1

// 8 players, 2/match example:
Match 1 winner → Round 2 Match 1  [ceil(1/2)=1]
Match 2 winner → Round 2 Match 1  [ceil(2/2)=1]
Match 3 winner → Round 2 Match 2  [ceil(3/2)=2]
Match 4 winner → Round 2 Match 2  [ceil(4/2)=2]

Total Rounds

totalRounds = log(players_per_match, max_players)

8  players, 2/match → log2(8)  = 3 rounds
16 players, 2/match → log2(16) = 4 rounds
8  players, 4/match → log4(8)  ≈ 2 rounds

10. Match Play Flow

━━━ Admin Bracket Generate karta hai ━━━
POST /api/v1/tournaments/{t}/generate-bracket
└─ Players randomly seed kiye jaate hain
└─ TournamentMatch rows ban jaate hain (saare rounds)
└─ TournamentMatchPlayer rows Round 1 ke liye
└─ Byes auto-handle (1 player vs bye → auto-win)
└─ tournament.status → in_progress

━━━ Player Match Room Claim karta hai ━━━
POST /api/v1/tournaments/{t}/claim-room
Response: {
  uuid: "tournament-match-5",
  mode: "tournament",
  max_players: 2,
  tournament_match_id: 5,
  bot_fill_after_seconds: 8,
  players: [{seat_no:1, user_id:101, display_name:"Arjun", ...}]
}
Match status: scheduled → waiting

━━━ Node.js Room Join ━━━
Unity → Node.js socket room "tournament-match-5"
8 seconds baad empty seats pe bots (agar bot_allowed=true)
Sab seats full → match start

━━━ Match Khatam ━━━
Node.js → tournamentMatchResultService.postResult({
  matchId, roomId, startedAt, endedAt,
  results: [{user_id:101, slot:1, score:4, finish_position:1, result:"win"},
            {user_id:202, slot:2, score:2, finish_position:2, result:"loss"}]
})

Retry: 3 times, exponential backoff (1s→2s→4s)
409/422 → no retry

━━━ Laravel Result Process ━━━
TournamentMatchResultController::submit()
├─ Match update: completed, scores, timestamps
├─ MatchPlayer update: score, finish_position, result
├─ Registration update:
│     winner (non-final round) → playing (aage khelega)
│     winner (final round)     → winner
│     losers                   → eliminated
├─ advanceBracket():
│     nextMatchNumber = ceil(match_number / players_per_match)
│     Winner ko next round ke match mein assign
└─ checkTournamentCompletion():
      Saare matches done? → completed → distributePrizes()

11. Knockout Bracket — Full Example (8 Players, 2/match)

BRACKET GENERATION (8 players, randomly seeded):
Seed: Arjun(1), Ravi(2), Priya(3), Suman(4), Rahul(5), Anjali(6), Meera(7), Vikram(8)

Round 1:  Match 1 → Arjun vs Ravi     | Match 2 → Priya vs Suman
          Match 3 → Rahul vs Anjali   | Match 4 → Meera vs Vikram

Round 2:  Match 1 → Winner(M1) vs Winner(M2)  [empty until R1 done]
          Match 2 → Winner(M3) vs Winner(M4)  [empty until R1 done]

Round 3:  Match 1 → Winner(R2M1) vs Winner(R2M2)  [FINAL]

BRACKET ADVANCEMENT (after Round 1 Match 2 completes, Priya wins):
  nextMatchNumber = ceil(2/2) = 1
  nextRound = 2
  → Priya added to Round 2, Match 1
  → If Match 1 already has 1 player → Match 1 status = waiting

BYE EXAMPLE (6 players → bracket size 8, 2 BYE slots):
  Match 3 → Rahul vs BYE → Rahul auto-wins → status=completed
  Match 4 → Meera vs BYE → Meera auto-wins → status=completed
  advanceByeWinners() runs → Rahul, Meera placed in Round 2

Total entry   = 8 × ₹100 = ₹800

Prize pool    = ₹800 × 80% = ₹640

1st (60%) = ₹640 × 0.60 = ₹384 → winner wallet credit
2nd (30%) = ₹640 × 0.30 = ₹192 → 2nd place wallet credit

13. Admin ka Workflow

TOURNAMENT APPROVE KARNA (user-created, entry_fee > ₹500):
Admin Panel: /admin/tournaments → Pending list
POST /admin/tournaments/{t}/approve → is_approved=true
Scheduler next minute mein registration window check karega

LIVE TOURNAMENT MANAGE:
1. Bots add karo (optional, max 5% of max_players):
   POST /{t}/add-bot {bot_difficulty:2, slot_count:2}

2. Registration manually close karo (ya auto-close hogi):
   POST /{t}/close-registration → prize amounts calculate

3. Bracket generate karo:
   POST /{t}/generate-bracket → matches ban jaate hain → in_progress

4. Kisi match ka result override karo (dispute):
   POST /internal/v1/tournaments/matches/{m}/override
   {winner_user_id: 101, reason: "Opponent disconnected maliciously"}

5. Tournament cancel:
   POST /{t}/cancel → RefundTournamentRegistrations job queue

6. Financial report:
   GET /{t}/financials
   → {total_entry_collected, platform_fee_20pct, prize_pool_80pct, prizes_paid, prizes_pending}

TOURNAMENT COMPLETION (automatic):
Last match result Node.js ne submit kiya
→ checkTournamentCompletion(): sab matches done
→ status=completed
→ distributePrizes(): wallets credit, platform fee log
