Master Plan

Ludo Social, Voice, AI Chat, and Friends System

This document combines the full roadmap for room chat, emoji, bot AI chat, Agora-based voice talk, friend requests, add-by-player-ID, and friend online or playing notifications.

Room Chat

Text chat, emoji, message history, reconnect-safe room communication.

Friends

Friend request from room and lobby, player-ID search, friend list, presence.

Voice

Temporary room voice channel using Agora with secure backend token issue.

Bot AI Chat

Safe controlled bot replies, templates first, AI-assisted later.

Main User Flows

  1. Players in same room can text chat and use emoji.
  2. If bot is in room, bot can also send short controlled chat.
  3. Players in same room can use live voice talk.
  4. Player can send friend request from room player card.
  5. Player can also send friend request from lobby by entering player ID.
  6. Later the app can notify when a friend comes online or starts playing Ludo.
Example: User A with player ID 12345678 sends a request to User B with player ID 78945612 from the room or from the lobby search box.

Product Modules

Module Includes

Recommended Release Phases

Phase 1: Emoji

Fast low-risk release with emoji tray, socket event, cooldown, and seat animation.

Phase 2: Text Chat

Add room messaging, history, Laravel persistence, reconnect support, and moderation basics.

Phase 3: Friend Requests

Add in-room friend action, lobby add-by-ID, accept and reject flow, and friend list.

Phase 4: Presence and Notifications

Add friend online, lobby, and in-match presence plus lightweight notifications.

Phase 5: Agora Voice Talk

Add backend token issue, Unity voice UI, room voice join and leave, and token renew flow.

Phase 6: Bot AI Chat

Start with safe templates, then move to AI-assisted short replies only if needed.

Architecture Decision

  • Reuse existing Ludo room identity for room chat and emoji.
  • Keep friends system separate at app level.
  • Use same room mapping for Agora voice channels.
  • Generate bot chat on server side, not in Unity.

Bot AI Chat Plan

Recommendation: do not start with fully free-form AI chat.
  1. Event-driven templates first
  2. Context-aware variations second
  3. Optional AI-assisted short replies later

Bot triggers can include join, emoji, capture, timeout, match start, and match end.

Bot replies should stay short, playful, non-toxic, and rate-limited.

Agora Voice Talk Plan

Agora is a practical RTC choice here because it supports low-latency room voice and secure token-based access.

Recommended channel naming

  • ludo_room_{roomUuid}
  • ludo_private_{roomUuid}
  • ludo_tournament_{roomUuid}

Required values

  • App ID
  • App Certificate
  • backend-generated RTC token
  • channel_name
  • agora_uid
Security: keep the App Certificate on the backend only.

Agora setup steps

  1. Create account in Agora Console.
  2. Create a project for Ludo voice.
  3. Copy the project App ID.
  4. Keep App Certificate enabled for token generation.
  5. Build Laravel token endpoint for room voice join.
  6. Unity fetches token from backend, not directly from Agora.
  7. Unity joins with App ID, channel_name, agora_uid, and rtc_token.
  8. When token is near expiry, Unity requests a fresh token and renews it.

Official Agora Note

This plan uses Agora official guidance checked on April 3, 2026: token-based authentication is the recommended secure approach, clients should fetch tokens from their own backend, and Unity voice integration follows the standard RTC channel-join flow.
